MARKETING INTELLIGENCE PLATFORM

INTERNAL_BUILD

GPT-4o text-to-SQL chatbot over 6 fragmented healthcare marketing sources — built solo in 4 weeks at Keck Medicine USC.

SYSTEM MOTIVATION

Marketing data at Keck was fragmented across 6 sources — ad spend, call volume, Salesforce leads, patient encounters, form submissions, digital performance — with no unified view. Every query required an analyst. The goal: let any stakeholder ask a plain-English question and get a chart in seconds, without touching SQL or filing a request. The chatbot turns a bottleneck into self-service.

SYSTEM OVERRIDES
  • → Mandatory CTEs for multi-source joins: ad spend × leads × Salesforce produces a cartesian product without explicit grouping. System prompt enforces CTEs for any query touching 2+ sources — prevents API hangs.
  • → Conversation memory with data snapshots: first 8 rows of each result passed back as JSON context, so "break that down by month" works correctly without re-stating filters from the previous turn.

FIN_RISK

BUSINESS CLOSURE RISK ENGINE

GITHUB ↗

Predicting SMB insolvency from 44.5M behavioral signals — alternative data applied to the financial early-warning problem.

SYSTEM MOTIVATION

No public dataset has revenue or credit data on small businesses at scale. But 44.5M Google Maps reviews exist — and customers vote with their attention. Declining review volume, worsening sentiment, owner disengagement are leading indicators of business health that surface before a closure appears in any official record. The same logic hedge funds use when buying satellite imagery of parking lots to predict quarterly earnings — applied to SMB credit risk.

SYSTEM OVERRIDES
  • → XGBoost over neural nets: tabular data with engineered features consistently favors gradient boosting. scale_pos_weight handled the 15:1 class imbalance without SMOTE-induced distribution distortion.
  • → Line-by-line streaming: 44M rows don't fit in RAM. Streaming forced a cleaner architecture — features computed incrementally, closer to how a production system would actually operate.

HONEST REFLECTION
"Businesses with 1–4 reviews produce statistically unreliable signals — a single bad review after a perfect run looks identical to genuine decline. The model was confidently wrong on sparse businesses. The 5-review floor wasn't a heuristic; it was the point where every signal became computable with reasonable variance."

REAL-TIME RECOMMENDATION

GITHUB ↗

Two-tower BPR retrieval on 857K reviews, 98K users — and how the capacity trap led to the learning rate fix.

SYSTEM MOTIVATION

Amazon Video Games was chosen deliberately: a review almost always means the person played it. Unlike books — reviewed by people who didn't finish them — game reviews are clean implicit feedback. The 99.97% sparsity is also realistic; most production recommendation systems are exactly this sparse, which is the actual challenge a retrieval architecture has to solve.

SYSTEM OVERRIDES
  • → BPR over BCE: BCE treats all unobserved pairs as negatives — wrong for 99.97% sparse data. BPR asks only: did the user prefer item A over B? That's the right framing for implicit feedback.
  • → The capacity trap: v1 used dim=64 and underperformed. v2 doubled to dim=128 and matched v1 exactly — both peaked at epoch 1 and collapsed. That ruled out capacity. The model was overshooting. Dropping LR to 3e-4 + ReduceLROnPlateau let it converge to epoch 38.

VISION TRANSFORMER FROM SCRATCH

REPO_PENDING

Measuring what inductive bias is worth — ViT from scratch vs. pretrained ResNet-18 on GTSRB traffic sign classification.

SYSTEM MOTIVATION

Traffic signs were chosen deliberately. A sign has a clear focal region — the symbol — surrounded by background clutter. If attention learns correctly, it should localize to that symbol with zero bounding box supervision. That makes the visualization legible: you can see exactly what the model attends to and judge whether it's learning the right thing. GTSRB's train/test domain shift (different lighting, angles, conditions) makes generalization an actual challenge, not a formality.

SYSTEM OVERRIDES
  • → Conv2d as patch embedding: mathematically equivalent to linear projection, but CUDA kernels make it 2–3× faster. No accuracy cost, free speedup.
  • → Attention stored on the module: weights saved as .attn_weights on each MHSA after every forward pass — no hooks, no extra code paths to break during visualization.

RESEARCH NOTE
"Val accuracy hit 99.47% — the architecture works. The 13-point val→test gap (99.47% → 86.25%) is what 'inductive bias' means in practice: ResNet's translation invariance handled new lighting and angles; ViT had to learn all of that from 24K images. Explaining the gap is more interesting than closing it with pretraining."
